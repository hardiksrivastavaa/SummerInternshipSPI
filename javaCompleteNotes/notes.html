Java components

  • JDK : Java Development Kit is a Software Pacakage. It contains Java Compiler, Java Runtime Environment and Java Virtual Machine.
  • Java Compiler : Java Compiler is a type of translator, which converts source code to byte code.
  • JRE : Java Runtime Environment create a layer between java application and operating system. Java application is executed under supervision of JRE.
  • JVM : Java Virtual Machine is an agent of java, which converts byte code to machine code.

Execution Scenario of Java

  • 1. Source Code (.java) Compiled in Byte Code (.class) through Java Compiler.
  • 2. Byte Code (.class) Executed through JVM.
  • 3. JVM Converts the byte in Machine Code and we Get Output.

What is OOPs? Write its pillars.

OOPs (Object-Oriented Programming) is a programming paradigm that organizes software design around objects, rather than functions or logic. The four main pillars of OOPs are:

Encapsulation : Bundling the data (variables) and methods (functions) that operate on the data into a single unit or class. It also restricts access to certain components.
Inheritance : Creating a new class from an existing class, inheriting its properties and behaviors, and possibly adding new ones.
Polymorphism : Allowing objects to be treated as instances of their parent class while also being able to behave differently based on their specific class type.
Abstraction : Hiding complex implementation details and showing only the essential features to the user.

What is Java? Write difference between C, C++ and Java.

Java is a high-level, class-based, object-oriented programming language that is designed to have as few implementation dependencies as possible. It was developed by Sun Microsystems (now owned by Oracle Corporation) and released in 1995. Java enables developers to write code that can run on any platform that supports Java without the need for recompilation, thanks to its "write once, run anywhere" (WORA) capability. Java applications are compiled to bytecode that can run on any Java Virtual Machine (JVM) regardless of the underlying computer architecture.

Why is Java Platform Independent?

Java is platform-independent because of how it compiles and runs code. Here’s a simple explanation:

  • Compilation to Bytecode:

    Java code is first compiled into an intermediate form called bytecode, which is the same for all platforms.

  • Java Virtual Machine (JVM):

    The bytecode runs on the JVM, which is different for each platform (Windows, Linux, etc.). The JVM translates the bytecode into machine code specific to that platform.

  • Write Once, Run Anywhere:

    Because the bytecode can run on any JVM, Java programs can run on any device or operating system that has a JVM, without needing to be rewritten or recompiled.

This is why Java is considered platform-independent.

Describe JDK, JVM and JRE.

JDK (Java Development Kit)

The JDK is a software package used by developers to build Java applications. It includes:

  • Compiler: Converts Java code into bytecode.
  • JRE: Provides the environment to run Java programs.
  • Tools: Includes tools like debuggers and profilers for development.

JRE (Java Runtime Environment)

The JRE is the software that provides the necessary environment to run Java programs. It includes:

  • JVM: Executes the Java bytecode.
  • Core Libraries: Provides the essential classes and resources for Java applications.

JVM (Java Virtual Machine)

The JVM is the part of the JRE that actually runs Java programs. It:

  • Executes Bytecode: Converts bytecode into machine code specific to the operating system.
  • Manages Memory: Handles tasks like garbage collection.
  • Ensures Platform Independence: Allows Java programs to run on any platform with a JVM.

What is `public static void main(String[] args)`?

The main method is the entry point of a Java application. It is where the Java runtime starts executing the program.

  • public: The method is accessible from anywhere, allowing the Java runtime to call it.
  • static: The method belongs to the class itself, not to instances of the class. This means it can be called without creating an instance of the class.
  • void: The method does not return any value.
  • main: The special name of the method that the Java runtime looks for as the entry point.
  • String[] args: An array of strings that can hold command-line arguments passed to the program.

Example

Here’s a simple example:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

In this example:

  • The main method prints "Hello, World!" to the console.
  • It serves as the starting point of the program's execution.

In summary, public static void main(String[] args) is the method where the Java program begins execution.

Difference Between `next()` and `nextLine()` Methods

The next() and nextLine() methods are used to read input from the user in Java, but they work differently:

`next()` Method

The next() method reads the next token (word) from the input. It stops reading when it encounters a space or a newline character.

  • Usage: Use next() to read a single word or token at a time.
  • Delimiters: It uses whitespace (spaces, tabs) as delimiters by default.
  • Example: 
                    import java.util.Scanner;
                
                public class Example {
                    public static void main(String[] args) {
                        Scanner scanner = new Scanner(System.in);
                        System.out.print("Enter your name: ");
                        String name = scanner.next();
                        System.out.println("Hello, " + name + "!");
                    }
                }

`nextLine()` Method

The nextLine() method reads the entire line of input until it encounters a newline character. It includes all characters, including spaces.

  • Usage: Use nextLine() to read a full line of text, including spaces.
  • Delimiter: It uses the newline character as the delimiter.
  • Example: 
                    import java.util.Scanner;
                
                public class Example {
                    public static void main(String[] args) {
                        Scanner scanner = new Scanner(System.in);
                        System.out.print("Enter your full name: ");
                        String fullName = scanner.nextLine();
                        System.out.println("Hello, " + fullName + "!");
                    }
                }

Summary

  • next(): Reads the next token (word) until whitespace.
  • nextLine(): Reads the entire line until newline.

Use next() for single words and nextLine() for full lines of input.

Difference Between `print()` and `println()` Methods

In Java, the print() and println() methods are used for displaying output to the console. Here’s how they differ:

`print()` Method

The print() method outputs text to the console without moving to a new line. It keeps the cursor on the same line after the output.

  • Usage: Use print() when you want to continue printing on the same line.
  • Example: 
    public class Example {
    public static void main(String[] args) {
        System.out.print("Hello, ");
        System.out.print("World!");
    }
}
  • Output: Hello, World!

`println()` Method

The println() method outputs text to the console and moves the cursor to a new line after the output. It is used to print text followed by a newline character.

  • Usage: Use println() when you want to print text and start a new line after it.
  • Example: 
    public class Example {
    public static void main(String[] args) {
        System.out.println("Hello,");
        System.out.println("World!");
    }
}
  • Output: Hello,
    World!

Summary

  • print(): Outputs text without moving to a new line.
  • println(): Outputs text and moves to a new line.

Use print() to continue on the same line and println() to start a new line after the output.

Java Program to Find the Area and Perimeter of a Rectangle

Below is a Java program that calculates the area and perimeter of a rectangle:

import java.util.Scanner;

public class Rectangle {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        // Input: Length and Width of the rectangle
        System.out.print("Enter the length of the rectangle: ");
        double length = scanner.nextDouble();
        
        System.out.print("Enter the width of the rectangle: ");
        double width = scanner.nextDouble();
        
        // Calculate area and perimeter
        double area = length * width;
        double perimeter = 2 * (length + width);
        
        // Output: Area and Perimeter
        System.out.println("Area of the rectangle: " + area);
        System.out.println("Perimeter of the rectangle: " + perimeter);
        
        scanner.close();
    }
}

This program performs the following steps:

  • Input: Reads the length and width of the rectangle from the user.
  • Calculation: Computes the area (length × width) and perimeter (2 × (length + width)).
  • Output: Displays the area and perimeter of the rectangle.

To run this program, compile it using a Java compiler and execute it. The program will prompt you for the dimensions of the rectangle and then display the results.

Java Program to Convert Days into Years, Weeks, and Days

Below is a Java program that takes the number of days as input and computes the equivalent years, weeks, and days:

import java.util.Scanner;

public class DaysConverter {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        // Input: Number of days
        System.out.print("Enter the number of days: ");
        int totalDays = scanner.nextInt();
        
        // Constants
        int daysInYear = 365;
        int daysInWeek = 7;
        
        // Calculate years, weeks, and days
        int years = totalDays / daysInYear;
        int remainingDays = totalDays % daysInYear;
        int weeks = remainingDays / daysInWeek;
        int days = remainingDays % daysInWeek;
        
        // Output: Years, Weeks, and Days
        System.out.println(totalDays + " days is equivalent to:");
        System.out.println(years + " year(s)");
        System.out.println(weeks + " week(s)");
        System.out.println(days + " day(s)");
        
        scanner.close();
    }
}

This program performs the following steps:

  • Input: Reads the total number of days from the user.
  • Calculation: Computes the number of years, weeks, and days by dividing and taking remainders.
  • Output: Displays the number of years, weeks, and days.

To run this program, compile it using a Java compiler and execute it. The program will prompt you for the number of days and then display the equivalent years, weeks, and days.
